# SECURESHIELD AI Database Schema (Firestore)

This document outlines the Firestore structure for the SECURESHIELD AI platform.

## Collections

### `scans`
Stores all vulnerability scan results generated by users.

| Field | Type | Description |
|-------|------|-------------|
| `userId` | `string` | The Firebase Auth UID of the user. |
| `url` | `string` | The target website scanned. |
| `platform` | `string` | Detected server/CMS (e.g., WordPress, Apache). |
| `securityChecks` | `map` | Boolean mapping of pass/fail for specific checks (HSTS, CSP, etc.). |
| `createdAt` | `timestamp` | Server-side timestamp for when the scan occurred. |
| `score` | `number` | Calculated overall security score (0-100). |
| `vulnerabilities`| `array` | List of detected issues with severity and details. |

## Access Rules
All data is protected by the rules in `/db/firestore.rules`.
- Reads are only allowed for the owner (`userId`).
- Anonymous sessions cannot write to the database (SaaS Protection).
